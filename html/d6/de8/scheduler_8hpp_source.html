117  class TaskModelCmp {
118  public:
119  /*
120  Remember that lower numbers mean higher priorities...
121  This function returns true if "a" has higher priority
122  than "b". This function was modified to work also in
123  EDF*:
124 
125  - when 2 tasks have the same priority (i.e. deadline in
126  EDF*), then the priority is decided basing upon the
127  insertion time (see setInsertTime() method).
128 
129  - when 2 tasks have the same priority and the same
130  insertion time, then the priority is decided basing
131  upon the task number (lower number => higher priority)
132 
133  It is possible to specify a "slice" time for each
134  task. This slice time models the quantum in round robin
135  schedulers. When a task starts executing, it is
136  assigned a timeout equal to the slice size. If the
137  timeout expires while the task is still executing, the
138  task is suspended, removed from the ready queue, and
139  re-inserted with a higher insertion time. In this way,
140  it is like inserting back in the queue of tasks with
141  the same priority.
142 
143  By using the slice time, it is possible to implement
144  the SCHED_RR policy of POSIX, by using a fixed
145  priority, and setting the slice time for each task.
146 
147  @todo check if this works with multi-processors (I
148  guess not).
149  */
150  bool operator()(TaskModel* a, TaskModel* b) const;
151  };
152  };
