64  class RTModelCmp {
65  public:
66  /*
67  Remember that lower numbers mean higher
68  priorities... This function returns true
69  if "a" has higher priority than "b". This
70  function was modified to work also in EDF*:
71 
72  - when 2 tasks have the same priority
73  (i.e. deadline in EDF*), then the priority
74  is decided basing upon the insertion time
75  (see setInsertTime() method).
76 
77  - when 2 tasks have the same priority and
78  the same insertion time, then the priority
79  is decided basing upon the task number
80  (lower number => higher priority)
81 
82  It is possible to specify a "slice" time
83  for each task. This slice time models the
84  quantum in round robin schedulers. When a
85  task starts executing, it is assigned a
86  timeout equal to the slice size. If the
87  timeout expires while the task is still
88  executing, the task is suspended, removed
89  from the ready queue, and re-inserted with
90  a higher insertion time. In this way, it is
91  like inserting back in the queue of tasks
92  with the same priority.
93 
94  By using the slice time, it is possible to
95  implement the SCHED_RR policy of POSIX, by
96  using a fixed priority, and setting the
97  slice time for each task.
98 
99  @todo check if this works with
100  multi-processors (I guess not).
101  */
102  bool operator()(RTModel* a, RTModel* b);
103  };
